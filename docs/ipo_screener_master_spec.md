# IPO Screener — Specification, Architecture & Implementation

---

## 1. Overview

A React web application that screens recent IPOs against a user-defined universe of tickers, displays live market data, and surfaces upcoming IPOs from a public calendar API. Data is refreshed automatically every day after market close via GitHub Actions, and can also be refreshed on demand from within the dashboard.

---

## 2. Goals

| Goal | Detail |
|---|---|
| Screen IPOs | Filter a user-supplied ticker universe against four criteria |
| Live market data | Price, volume, market cap, P/E, and price changes fetched from Yahoo Finance |
| Upcoming IPOs | Separate panel sourced from Finnhub's free IPO calendar API |
| Daily automation | GitHub Actions job runs at 4:30 PM ET every trading day |
| Manual refresh | Button in the dashboard triggers the same fetch on demand |
| Export | Download filtered results as CSV |

---

## 3. Screener Criteria

All four filters are **pre-filled with defaults but adjustable in the UI**.

| Filter | Default | Adjustable |
|---|---|---|
| Market Cap | > $2,000,000,000 | Yes |
| Average Volume (30-day) | > 1,000,000 | Yes |
| IPO Date | Within last 365 days from today | Yes (day range) |
| EPS Growth Next Year | > 0% | Yes |

Missing data is handled **leniently**: stocks with unavailable values are shown with an `N/A` badge and are **excluded** from filter matching (they will not pass a filter they cannot be evaluated against).

## 5. Upcoming IPOs Panel

A secondary panel on the dashboard (separate from the screener table) showing IPOs scheduled in the next 30 days.

| Field | Source |
|---|---|
| Company Name | Finnhub |
| Expected Ticker | Finnhub |
| Exchange | Finnhub |
| Expected IPO Date | Finnhub |
| Price Range | Finnhub |
| Shares Offered | Finnhub |

**Data source:** `GET https://finnhub.io/api/v1/calendar/ipo?from=YYYY-MM-DD&to=YYYY-MM-DD&token=API_KEY`

**Coverage:** US exchanges (NYSE, NASDAQ) only on the free tier. International upcoming IPOs are not available without a paid plan.

**Refresh cadence:** Fetched as part of the same daily GitHub Actions job and manual refresh cycle.

**API key requirement:** A free Finnhub account is required. The key is stored as a GitHub Actions secret (`FINNHUB_API_KEY`) and is never committed to the repository.

**Key design decision — no backend server:** All data lives as static JSON files in the repo. The React app fetches them directly from GitHub Pages. This means zero hosting cost and zero server maintenance. The GitHub Actions job is the only "backend."

## 9. Implementation Plan

### Step 1 — Repository Setup

1. Create a new GitHub repository (e.g. `ipo-screener`).
2. Add `universe.csv` with a `Ticker` header and one ticker per row.
3. Enable GitHub Pages from the `/docs` folder on the `main` branch.
4. Add repository secrets:
   - `FINNHUB_API_KEY` — free key from [finnhub.io](https://finnhub.io)
   - `GH_PAT` — a GitHub Personal Access Token with `repo` write scope (so Actions can push `data.json` back)

---

### Step 2 — Fetch Script (`scripts/fetch_data.py`)

```python
import yfinance as yf
import requests
import json
import csv
import os
from datetime import datetime, timedelta

FINNHUB_KEY = os.environ["FINNHUB_API_KEY"]

# --- Load tickers ---
tickers = []
with open("universe.csv") as f:
    reader = csv.DictReader(f)
    for row in reader:
        tickers.append(row["Ticker"].strip())

# --- Fetch market data ---
results = []
for ticker in tickers:
    try:
        info = yf.Ticker(ticker).info
        hist = yf.Ticker(ticker).history(period="1mo")

        price = info.get("currentPrice") or info.get("regularMarketPrice")
        prev_close = info.get("previousClose")
        week_ago_price = hist["Close"].iloc[-6] if len(hist) >= 6 else None
        month_ago_price = hist["Close"].iloc[0] if len(hist) >= 1 else None

        def pct_change(current, base):
            if current and base and base != 0:
                return round((current - base) / base * 100, 2)
            return None

        results.append({
            "ticker": ticker,
            "name": info.get("longName") or info.get("shortName"),
            "ipoDate": info.get("ipoExpectedDate") or info.get("firstTradeDateEpochUtc"),
            "marketCap": info.get("marketCap"),
            "avgVolume": info.get("averageVolume"),
            "epsGrowthNextYear": info.get("earningsGrowth"),
            "price": price,
            "change1d": pct_change(price, prev_close),
            "change1w": pct_change(price, week_ago_price),
            "change1m": pct_change(price, month_ago_price),
            "sector": info.get("sector"),
            "industry": info.get("industry"),
            "peRatio": info.get("trailingPE"),
        })
    except Exception as e:
        results.append({"ticker": ticker, "error": str(e)})

with open("data.json", "w") as f:
    json.dump({
        "lastUpdated": datetime.utcnow().isoformat() + "Z",
        "stocks": results
    }, f, indent=2)

# --- Fetch upcoming IPOs from Finnhub ---
today = datetime.utcnow().date()
in_30 = today + timedelta(days=30)
url = (
    f"https://finnhub.io/api/v1/calendar/ipo"
    f"?from={today}&to={in_30}&token={FINNHUB_KEY}"
)
resp = requests.get(url, timeout=10)
ipos = resp.json().get("ipoCalendar", []) if resp.ok else []

with open("upcoming_ipos.json", "w") as f:
    json.dump({
        "lastUpdated": datetime.utcnow().isoformat() + "Z",
        "ipos": ipos
    }, f, indent=2)

print(f"Done. {len(results)} stocks fetched. {len(ipos)} upcoming IPOs.")
```

### Step 3 — GitHub Actions Workflow (`.github/workflows/fetch.yml`)

```yaml
name: Fetch Market Data

on:
  schedule:
    - cron: "30 21 * * 1-5"   # 4:30 PM ET (21:30 UTC), Monday–Friday
  workflow_dispatch:            # Manual trigger from GitHub UI or dashboard

jobs:
  fetch:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          token: ${{ secrets.GH_PAT }}

      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - run: pip install yfinance requests

      - run: python scripts/fetch_data.py
        env:
          FINNHUB_API_KEY: ${{ secrets.FINNHUB_API_KEY }}

      - name: Commit and push updated data
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add data.json upcoming_ipos.json
          git diff --cached --quiet || git commit -m "chore: update market data $(date -u +%Y-%m-%dT%H:%M:%SZ)"
          git push
```

> **Manual refresh from the dashboard** calls the GitHub Actions API (`POST /repos/{owner}/{repo}/actions/workflows/fetch.yml/dispatches`) using a limited-scope PAT stored as an environment variable in the built app. The dashboard polls `data.json`'s `lastUpdated` field to detect when new data has landed.

### Step 4 — React App

**Tech stack:** React + Vite + Tailwind CSS. No backend. Deployed to GitHub Pages via the `/docs` build output.

**Key components:**

- **`FilterBar`** — Four numeric inputs (Market Cap floor, Volume floor, IPO date window in days, EPS Growth floor). All pre-filled with agreed defaults. Filters apply client-side in real time.
- **`ScreenerTable`** — Sortable by any column. Paginated (25 rows per page). Colour-coded positive/negative price changes. `N/A` badge for missing values.
- **`UpcomingIPOs`** — Card-style panel below the main table. Shows next 30 days of IPOs from Finnhub.
- **`ExportButton`** — Exports the currently **filtered** result set (not the full universe) to CSV.
- **`RefreshBadge`** — Shows `Last updated: X minutes ago`. Includes a "Refresh Now" button that triggers the GitHub Actions workflow dispatch.

## 11. Known Limitations

| Limitation | Detail |
|---|---|
| EPS Growth Next Year | Sparsely populated by Yahoo Finance, especially for small-caps and international stocks — expect many `N/A` values |
| International stocks | `yfinance` covers many international exchanges (LSE, TSX, ASX, etc.) via suffix (e.g. `HSBA.L`) but data quality and completeness is lower than for US tickers |
| Upcoming IPOs coverage | Finnhub free tier covers US exchanges only; international upcoming IPOs require a paid plan |
| IPO Date field | Yahoo Finance does not expose a clean IPO date for all tickers; some may show `N/A` |
| Rate limits | `yfinance` is an unofficial scraper — large universes (200+ tickers) may hit throttling; the script includes a small delay between calls |
| Real-time prices | Data is end-of-day, not real-time intraday |
